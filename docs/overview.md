# 喜马拉雅音频下载器 · 项目概览

> Ximalaya-Downloader-Next　|　状态：开发中
> 一个稳定、可扩展、便于长期维护的喜马拉雅音频下载工具。

## 简介

喜马拉雅官方没有提供将音频保存到本地的功能。本项目提供命令行、终端面板与可复用的 Python 库，帮助你把**自己有权访问的**音频（免费内容、已购或会员可听内容）下载并整理到本地，支持单曲与整张专辑、断点续传、失败重试与优雅停止。

整体采用「核心库 + 薄前端」结构：能力沉淀在独立、可测试、可扩展的核心库中，命令行与终端面板都只是其上的一层薄壳，未来的图形界面、Web 或 HTTP API 都可复用同一套核心。**安装与使用见 [`README.md`](../README.md)**，架构与设计原则见 [`architecture.md`](./architecture.md)。

## 核心特性

- **单曲与专辑下载**：按链接或 ID 下载单个音频，或整张专辑、专辑中的指定区间。
- **音质选择**：在可用音质之间选择，并在目标音质缺失时自动回退。
- **断点续传**：文件级（已存在跳过）+ 字节级（`.part` 断点续传，带 `If-Range` 内容校验）+ 任务级（`xdl resume` 从任务库恢复）。
- **优雅停止**：下载中 `Ctrl-C`（或 TUI「停止」）会存好进度再退出，可随后 `resume` 继续。
- **可靠的下载管理**：任务状态持久化到本地 SQLite（`~/.xdl/tasks.db`），失败按类型分级重试 + 失败收尾轮。
- **克制的请求策略**：受保护播放信息默认串行；遇到风控立即熔断整批并留下最小化观测记录，不自动持续重试。
- **登录与凭据管理**：接管真实 Chrome 登录并把会话持久化在专用配置目录，用于访问会员/已购内容。
- **多形态前端**：命令行 `xdl`、终端面板 `xdl-tui`，以及可在脚本中导入的 Python 库（`from xdl import Facade`），共用同一套核心。

> 安装、CLI/TUI/库的具体用法见 [`README.md`](../README.md)。

## 项目结构（顶层）

```
ximalaya-downloader-next/
├─ src/xdl/         核心库（领域 / 接口 / 应用 / 适配器 / 配置 / 前端）
├─ docs/            项目文档（本概览 + 架构设计）
├─ tests/           单元 / 契约测试
├─ downloads/       默认下载输出目录
└─ pyproject.toml
```

`src/xdl/` 内部的分层与模块划分见架构文档；前端位于 `src/xdl/frontends/`（`cli.py` 与 `tui.py`）。

| 阶段 | 内容 | 状态 |
|---|---|---|
| M0 | 项目骨架：接口定义、领域模型、配置骨架、测试框架 | ✅ 已完成 |
| M1 | 请求鉴权子系统打通（默认实现 + 兜底实现 + 自动降级） | ◐ MVP 采用「让页面自身生成签名」方案；因平台对自动化加风控，改为 **CDP 接管真实 Chrome**（见架构 §7.1 备注），本地签名/降级链待做 |

| M4 | 登录与凭据管理：多登录方式、会员/已购内容 | ◐ 已支持登录 + 会话持久化（专用 Chrome 配置目录）；多方式待扩展 |
| M5 | 专辑增量同步 + 搜索 | ☐ 待做 |
| M6 | 本地文件解码备用通道 | ☐ 待做 |
| M7 | 打包分发（pip + 单文件可执行）+ 自动更新检查 + 文档完善 | ◐ 已可 `pip install -e .`；单文件分发待做 |
| M8 | 图形界面 | ◐ 已提供 Textual **终端面板（TUI）** `xdl-tui`（实时任务表 + 下载/恢复/停止）；桌面 GUI 待做 |

> **当前实现**：CLI 提供 `xdl login`、`xdl track`（单曲）、`xdl album`（专辑有界并发批量下载，
> 支持 `--range` 区间与 `--quality` 音质）与 `xdl resume`（从任务库恢复未完成任务）；另有终端面板 `xdl-tui`。
> 单曲与专辑逐集均纳入 SQLite 任务库，支持文件级/字节级/任务级续传与优雅停止。
> `search`/`config` 等命令为路线图目标，尚未实现。
> 免登录下载暂不支持（匿名请求会被风控 `1001` / 地区限制 `927` 拦截）；但**专辑曲目清单**走免签公开接口，
> 无需登录即可获取（逐集下载仍需登录）。
